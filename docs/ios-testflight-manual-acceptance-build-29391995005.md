## 2. 如何填写

结果统一填写以下四种状态：

- `PASS`：实际结果与预期一致。
- `FAIL`：可以完成操作，但结果错误、缺失或体验明显异常。
- `BLOCKED`：因为闪退、卡死、权限、账号或前置数据问题无法继续。
- `SKIP`：当前账号没有对应数据，或该条件不适用。

执行规则：

1. 先完成“P0 冒烟测试”。任一 P0 出现闪退、数据丢失、金额错误或无法登录时，先停止破坏性测试。
2. 新建测试数据统一在标题或备注中加入 `TF29391995005`，便于测试后清理。
3. 删除、归档、还款和账户调整只操作专门创建的测试数据。
4. 每个失败至少保留一张截图；涉及加载、闪退或跳转错误时，建议提供操作前、操作中、操作后三张。
5. 性能时间可以用主观区间记录：`<1 秒`、`1-3 秒`、`3-5 秒`、`>5 秒`。
