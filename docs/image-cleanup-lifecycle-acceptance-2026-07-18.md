# 原图生命周期与异步删号验收记录（2026-07-18）

## 已上线契约

- Supabase migration：`069` 至 `077`。
- `ingest-receipt`：生产版本 `v160`。
- 留存扫描：每小时创建到期清理任务。
- 清理 worker：每 5 分钟执行一次，并优先推进一个待删除账号。
- 账号删除状态：`requested -> cleaning -> deleting -> completed`，失败可恢复为 `requested` 或 `failed` 后重试。
- 新路径：`user-id/YYYY-MM-DD/hash.jpg` 与 `tmp/user-id/...`。
- 历史路径：`YYYY-MM-DD/hash.jpg`，只有所有现存引用都属于同一用户时才允许删除。
- Storage 删除成功后才清空数据库引用；`storage_deleted_at` 支持中断后断点续跑。
- 清理任务进入 `processing` 后，任何用户都不能再给同一路径增加新引用。
- 账号进入删除流程后，upload token 被清空，旧 JWT 上传也返回 `410`。

## PWA 手动验收

1. 在测试账号把“截图原图留存”设为“不保留”，上传一张新图；识别结果应保留，刷新后原图不再可见，队列最终归零。
2. 从“永久”切到有限期限并选择“立即清理”；提示应显示已删除、跳过或剩余数量，不应统一显示成功。
3. 删除一条带图的支出、收入和通用域记录；记录立即消失，图片清理失败时进入队列重试。
4. 用一条历史路径记录执行立即清理；同一路径没有其他用户引用时应清除图片和引用。
5. 只用临时测试账号执行删除账户；前端退出登录后，原登录凭据和 upload token 都不能继续上传。

## iOS 真机验收

1. 设置页切换“不保留 / 7 天 / 30 天 / 永久”，重新进入设置后选择保持一致。
2. “立即清理已有原图”显示删除数、剩余数和跳过数；部分失败不能显示成全部成功。
3. 清理后返回首页、记录详情和中转站，结构化记录仍存在，图片按留存结果显示为空或不可用。
4. 删除带图的消费、收入和通用域记录，随后刷新 PWA，跨端记录与图片状态一致。
5. 只用临时测试账号点击删除账户；收到后台处理中响应时 App 应立即退出登录，并显示“原图清理完成后自动删除全部数据”。
6. 删除请求提交后，快捷指令旧 token 应明确失败，不能继续写入原账号。
